# Медиация — процесс споров и апелляций (структура, не власть)

> Артефакт жизненного цикла спора/апелляции и его машинная проверка. Реализует
> механизм №6 **«Апелляции»** из [`docs/ANTI-ABUSE.md`](../../docs/ANTI-ABUSE.md):
> у каждого участника есть **прозрачный путь обжалования** отказа или санкции.
> **Решают люди**, не ИИ (Конституция [ст. 9.2](../../docs/CONSTITUTION.md)).

## Зачем это

Право на апелляцию в `ANTI-ABUSE.md` описано словами. Чтобы оно не осталось
лозунгом и не «разъехалось» с конституцией молча, процесс зафиксирован как
проверяемый артефакт [`dispute-process.json`](dispute-process.json), а
**Mediator AI-агент** ([`ai-agents/mediator_agent.py`](../../ai-agents/mediator_agent.py))
на каждый push/PR доказывает, что процесс по-прежнему соответствует конституции.

**Mediator структурирует спор, но НИКОГДА его не решает.** Агент не наказывает,
не оправдывает и не двигает средства — он только проверяет форму процесса и
докладывает «зелёно/красно». «Красный» — сигнал сообществу, а не действие.

## Жизненный цикл спора/апелляции

Артефакт описывает конечный автомат стадий:

| Стадия | Роль | Кто ведёт | Срок |
|---|---|---|---|
| `intake` (старт) | structure | Mediator-агент (структурирует обращение) | 3 дня |
| `review` | **decide** | независимые проверяющие, **≥2 людей** | 7 дней |
| `escalation` | **decide** | совет хранителей (мультисиг), **≥3** | 14 дней |
| `resolved` (терминал) | record | фиксация решения в публичном реестре | — |

Решение принимают только люди и только коллективно (≥2). Mediator-агент допустим
исключительно на не-решающих стадиях (приём `intake` и запись `resolved`).

## Санкции с правом обжалования

Каждая санкция из `ANTI-ABUSE.md` обязана иметь путь обжалования:

- `disbursement-rejection` — отказ в выплате/помощи;
- `reputation-sanction` — снижение репутации (№7);
- `temporary-freeze` — временная заморозка выплат (№8);
- `exclusion` — исключение участника (№9).

Апелляцию рассматривает **не тот**, кто вынес санкцию (независимость, §3).

## Что проверяет агент (6 проверок)

| Проверка | Что гарантирует | Норма |
|---|---|---|
| `appeal-for-every-sanction` | у каждой канонической санкции есть обжалование | §6 |
| `mediator-not-decider` | решают люди (≥2), не ИИ/медиатор/один человек | ст. 9.2 / §3 |
| `independent-review` | апелляцию решает не автор санкции | §3 |
| `valid-lifecycle` | один старт, есть терминал, нет тупиков/сирот | ст. 3 |
| `bounded-timelines` | у каждой не-терминальной стадии задан срок > 0 | ст. 3 / §6 |
| `process-links` | все doc-ссылки процесса ведут к реальным файлам | ст. 3 |

## Как запустить

```bash
python3 ai-agents/mediator_agent.py          # человекочитаемый отчёт
python3 ai-agents/mediator_agent.py --json    # машиночитаемый (для CI/агентов)
python3 ai-agents/test_mediator.py            # тест-инвариант: «красное ловится»
```

Зависимостей нет (стандартная библиотека Python), сети нет — детерминированно в
любом CI. **TESTNET-ONLY:** реальных средств/ключей/mainnet агент не касается.

## Рельсы

- ИИ не орган власти и ничем не владеет (Конституция ст. 9). Mediator **не решает**.
- Решение спора — всегда у независимого коллектива людей, никогда у одного лица.
- Всё открыто: процесс, проверка и итоги решений публичны (реестр решений).
- Приватность: публичен факт/адрес, не личность участника.

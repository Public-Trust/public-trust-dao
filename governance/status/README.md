# Статус-светофор проекта

## Главное — простыми словами

В этой папке лежит один файл — `run_all_status.json`. Это «светофор» проекта:
короткая машиночитаемая сводка последней самопроверки. Зелёно — все восемь
служебных ИИ-помощников и проверки качества в порядке; красно — что-то нужно
поправить.

Файл делает не человек, а команда самопроверки
([`ai-agents/run_all.py`](../../ai-agents/run_all.py)): она прогоняет всех
помощников разом и складывает их ответы в один короткий итог. Так любой человек
или дашборд может одним взглядом увидеть состояние проекта — **без внешних
сервисов, без сторонних бейджей, без слежки**. Всё открыто и проверяемо.

Важно: это **сигнал, а не власть** (Конституция, ст. 9). Файл ничего не решает и
ничем не распоряжается — он только показывает, что увидела самопроверка. Власть —
у голосования сообщества и Safe-мультисига.

## Что внутри файла

- `verdict` — общий итог: `green` (всё в порядке) или `red` (есть нарушение).
- `agents` — сколько из восьми ИИ-помощников зелёные (`green` из `total`).
- `tests` — сколько тест-инвариантов прошло (доказывают, что «красное» реально
  ловится, а не «зелёно по умолчанию»).
- `guard` — сводка стража структуры: его итог, счёт и **мягкие предупреждения**
  (число и сами строки — что именно стоит поправить, не валя сборку).
- `agent_status` — по строке на каждого помощника: его ключ и зелёно/красно.

## Как обновляется

```
python3 ai-agents/run_all.py --with-tests --status-out governance/status/run_all_status.json
```

Файл **детерминированный**: при неизменном состоянии репозитория он побайтово тот
же. В нём намеренно НЕТ времени по стенным часам — иначе он «грязнел» бы в git при
каждом прогоне. «Когда» берётся из времени коммита, а не из самого файла. Поэтому
артефакт меняется в истории ровно тогда, когда меняется сам вердикт или счёт.

Поле `schema_version` версионирует формат: если формат изменится несовместимо,
номер вырастет, и читатели смогут подстроиться.

## Технический раздел — для разработчиков

- Артефакт собирает `build_status(report)` из полного `run_all --json`-отчёта;
  пишет `write_status(path, report)` (создаёт родительские каталоги, финальный
  `\n`, `ensure_ascii=False`, `indent=2`).
- Полнота сведения доказана тест-инвариантом
  [`test_run_all.py`](../../ai-agents/test_run_all.py): структура, детерминизм
  (повторный прогон → побайтово тот же файл), запись и при `verdict=red`,
  устойчивость к отсутствию стража (`guard=None`).
- Это основа будущего публичного «статус-индикатора» без внешних сервисов:
  дашборд/страница могут читать этот файл напрямую из репозитория.
- TESTNET-only: артефакт не трогает средства/ключи и не выходит в сеть.

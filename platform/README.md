# Платформа Public Trust DAO (Next.js)

**Простыми словами.** Это рабочее приложение фонда — место, где человек сможет
попросить помощь, видеть открытый журнал решений, голосовать наравне со всеми и
проверять состояние общей казны. Мы строим её шаг за шагом: каждый готовый экран
сразу появляется на сайте.

Это **общественное благо, а не вложение денег.** Здесь не обещают доход, нет
пирамиды и нет платы за привлечение людей. Всё пока на **тестовой сети**, без
реальных средств.

## Почему Next.js и «статическая сборка»

Платформа написана на **Next.js (React)** — это полноценная основа, на которой
можно построить большое приложение. Пока мы собираем её как **набор обычных
файлов** (статическая сборка, `next export`), чтобы выкладывать бесплатно на
GitHub Pages без платного сервера. Архитектура при этом полноценная: когда
понадобится сервер, тот же код можно запустить как сервер, убрав режим
статической сборки.

## Готовые экраны

- **Главная** — миссия, дисклеймер «общественное благо, не инвестиция», витрина
  будущих экранов.
- **Подключить кошелёк** (`/wallet/`) — человек подключает кошелёк (например,
  MetaMask) с понятным объяснением «что это и зачем», видит свой публичный адрес
  и сеть, может отключиться. Деньги при подключении не двигаются; если выбрана
  основная сеть с реальными деньгами — экран честно предупреждает, что фонд
  работает только в тестовой сети.
- **Подать заявку на помощь** (`/apply/`) — спокойная анонимная форма по порядку
  важности нужд; обязательное поле целевого расхода (фонд оплачивает нужду
  напрямую). Пока контракты не в тестовой сети — собирает проверяемый черновик
  заявки прямо в браузере (скопировать/сохранить), ничего не уходит наружу.
- **Открытый журнал** (`/journal/`) — список всех решений фонда, новые сверху, с
  фильтром по виду записи и поиском. Простыми словами объяснена защита от
  подмены (сцепление записей отпечатками), показан общий отпечаток журнала,
  каждая запись ведёт на полный файл в репозитории. Данные едут вместе с
  платформой снимком из открытого реестра (`lib/journal-data.json`) — ничего не
  подгружается со стороны.
- **Голосование** (`/voting/`) — главное правило простыми словами: один человек —
  один голос, голос нельзя купить или продать. Список предложений; по открытому
  можно проголосовать «За» / «Против» / «Воздержаться», вес голоса всегда равен
  одному. Пока контракты не в тестовой сети — голос собирается проверяемым
  черновиком прямо в браузере (скопировать/сохранить/изменить) и хранится только
  у человека; на экране показательные предложения, ничего не уходит наружу.
- **Окно казны** (`/treasury/`) — состояние общей казны простыми словами и только
  для просмотра: сколько всего, сколько в неприкосновенном запасе, сколько на
  прямую помощь людям (всегда не меньше 70% распределяемого остатка) и на скромную
  благодарность за работу (под потолком); понятная полоска состава, уровень
  «насколько фонд обеспечен» и последние движения (поступления и целевые выплаты).
  Потратить отсюда нельзя; пока контракты не в тестовой сети — показательное
  состояние, суммы в тестовых единицах, ничего не подгружается со стороны.
- **Проверка личности** (`/identity/`) — простым языком, как фонд убеждается, что
  за участником один настоящий живой человек, без слежки и без сбора лиц в базу.
  Два слоя: сколько проверки нужно (зависит от действия — смотреть сайт без
  проверки, подать заявку — лёгкая проверка «живой человек», голосовать и получать
  помощь — строгая проверка уникальности) и каким способом проверяться (круг
  знакомых без биометрии / паспорт из множества признаков / проверка «живости» у
  внешнего сервиса / обязательный запасной путь — поручительство живых людей).
  Жёсткая граница «уникальность — не власть над деньгами». Пока проверка не
  запущена — выбор остаётся проверяемой памяткой в браузере без личных данных,
  ничего не уходит наружу.

Этим закрыта запланированная оператором серия экранов ценности (INBOX #34).

## Рельсы прозрачности (соблюдаются буквально)

- **Никаких сторонних трекеров, рекламы и аналитики.** Ноль запросов к чужим
  серверам.
- **Шрифты системные** — ничего не подгружается со стороны.
- **Зависимости зафиксированы** точными версиями (защита цепочки поставок),
  пакетов — минимум.
- **Телеметрия Next.js выключена** (`NEXT_TELEMETRY_DISABLED=1`).
- **Двуязычность RU↔EN** прямо в интерфейсе (переключатель в шапке).
- **Доступность:** ссылка «к содержимому», видимые кольца фокуса, светлая и
  тёмная темы, уважение к `prefers-reduced-motion`.

## Как запустить локально (для разработчиков)

```bash
cd platform
npm install
npm run dev      # режим разработки на http://localhost:3000
npm run build    # статическая сборка в out/
```

`npm run typecheck` — проверка типов без сборки.

Для публикации под подкаталог GitHub Pages задайте базовый путь:

```bash
NEXT_PUBLIC_BASE_PATH=/public-trust-dao npm run build
```

## Сборка и проверка

Локально на хосте сейчас не хватает места на диске для `node_modules`, поэтому
сборку проверяет рабочий процесс GitHub Actions
[`.github/workflows/platform.yml`](../.github/workflows/platform.yml): он ставит
зависимости, проверяет типы и собирает статический сайт на серверах GitHub.
Публикацию на Pages подключим отдельно, когда начнём переводить людей с
сайта-витрины [`web/`](../web/) на платформу.

## Структура

```
platform/
  app/            экраны и общий каркас (layout, globals.css)
    wallet/       экран «Подключить кошелёк»
    apply/        экран «Подать заявку на помощь»
    journal/      экран «Открытый журнал»
    voting/       экран «Голосование»
    treasury/     экран «Окно казны»
    identity/     экран «Проверка личности»
  components/     общие части интерфейса (шапка, подвал, провайдер языка/темы)
  lib/            двуязычные тексты (i18n) и слои данных (wallet.ts, application.ts,
                  journal.ts, voting.ts, treasury.ts, identity.ts) + снимок реестра (journal-data.json)
  next.config.mjs настройки сборки (статический экспорт)
  package.json    зависимости (точные версии)
```

**Разделение слоёв** (задел под будущий сервер): экраны (`app/`) и общий UI
(`components/`) отделены от данных и блокчейна (`lib/`). Например, `lib/wallet.ts`
— это чистый доступ к кошельку по стандарту EIP-1193 без сторонних библиотек;
экран его только использует. Так позже можно подключить настоящие контракты или
серверную часть, не переписывая интерфейс.

См. также: [`docs/PRODUCT-INTERFACES.md`](../docs/PRODUCT-INTERFACES.md) —
очередь интерфейсов (web → Telegram → мобильное).

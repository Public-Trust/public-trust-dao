- **2026-06-22 (сессия 4, Этап 3 — прозрачность, часть 2/2 ✅):**
  - `scripts/ipfs_manifest.py` — детерминированный IPFS content-манифест без
    сторонних зависимостей. Вычисляет **настоящие** IPFS CIDv1 (raw, single-block)
    локально, прямо из содержимого: для файла ≤256 KiB CID = base32(0x01 0x55 +
    sha2-256). Подтверждено эталоном (CID пустого файла совпал с канонической
    константой). Значит публичные IPFS-ссылки доступны и проверяемы БЕЗ аккаунта
    пиннера — аккаунт нужен только для persistence.
  - `governance/ipfs/manifest.json` — 6 ключевых файлов (`docs/*.md` +
    `governance/registry/index.json`) с size/sha256/CID + `manifest_fingerprint`.
  - `governance/ipfs/README.md` — как устроен CID и два способа независимой
    проверки (через `ipfs add` и чистым пересчётом `verify`).
  - CI `.github/workflows/ipfs.yml` — `verify` на push/PR: контент и CID не
    разойдутся незаметно.
  - Проверено «до зелёного»: build→verify=OK (6 записей); подмена документа →
    verify exit=1 (поймал size/sha256/CID/fingerprint), восстановление → exit=0.
  - Команды пиннинга для оператора: `python3 scripts/ipfs_manifest.py pin-commands`.

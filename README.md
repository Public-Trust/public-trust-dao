# Public Trust DAO

**Открытая, децентрализованная система взаимопомощи и общественных благ.**

Public Trust DAO обеспечивает людям базовую безопасность, человеческое достоинство, жильё, питание, медицинскую помощь, образование и восстановление самостоятельности — и возможность, получив помощь, со временем помогать другим.

> ⚠️ **Это НЕ инвестиция, НЕ финансовая пирамида, НЕ инструмент извлечения прибыли.** Это открытая инфраструктура взаимопомощи, управляемая конституцией, прозрачной блокчейн-казной, открытым кодом и сообществом.

## Главный принцип
Никто не является владельцем проекта. Основатель — обычный участник. Проект принадлежит своей **миссии, конституции и сообществу**. Люди и ИИ — временные хранители миссии.

- [`docs/PROOF-OF-CONTRIBUTION.md`](docs/PROOF-OF-CONTRIBUTION.md) — доказательство вклада и эскроу-контракты: фонд создаёт оплачиваемую работу через помощь людям; как контракт доказуемо подтверждает выполнение (подпись получателя + артефакты-хеши + независимые валидаторы со ставкой репутации) перед выплатой; честно о проблеме оракула.
- [`docs/IDENTITY-VERIFICATION.md`](docs/IDENTITY-VERIFICATION.md) — как убедиться, что за аккаунтом живой человек, без слежки: один человек = один аккаунт; фонд не собирает базу лиц (проверку делает внешний сервис, у нас остаётся только пометка «проверен»); в блокчейн — только факты, без личных данных; доказательство без раскрытия личности; готовые решения proof-of-personhood на выбор; обязательный запасной путь без камеры (поручительство вживую); строгость проверки по риску.
- [`docs/ACCOUNTABILITY.md`](docs/ACCOUNTABILITY.md) — принцип «всё под подписью и прослеживаемо»: у каждого значимого действия (участника, ИИ, разработчика) есть автор, оно подписано и записано без возможности подмены; деньги/власть/договорённости дополнительно закрепляются отпечатком в блокчейне (личные данные — нет); ИИ всегда под подписью, но без единоличной власти.
- [`LAUNCH.md`](LAUNCH.md) — техническое задание на построение инфраструктуры.
- [`contracts/`](contracts/) — смарт-контракты (TESTNET first). Этап 5, каркас:
  [`Treasury.sol`](contracts/contracts/Treasury.sol) — базовый слой казны (release
  только через executor=мультисиг/Timelock, лимит выплаты, аварийная пауза, события)
  + тесты «до зелёного». Реальных средств/ключей нет.
- `web/` — публичный сайт (GitHub Pages).
- `governance/` — управление (Safe multisig 3-из-5, Snapshot/Aragon).
  - [`governance/registry/`](governance/registry/) — публичный, проверяемый,
    защищённый от подмены реестр решений (append-only hash-chain).
  - [`governance/safe/`](governance/safe/) — макет тестовой казны Safe multisig
    3-из-5 (testnet, без реальных денег/ключей) + валидатор рельс.
  - [`governance/snapshot/`](governance/snapshot/) — макет off-chain голосования
    Snapshot (1 человек = 1 голос, не плутократия) + валидатор рельс.
- [`ai-agents/`](ai-agents/) — AI-модули, служащие миссии (не владеют средствами,
  не имеют власти; ст. 9). Этап 6, каркас: **Audit-агент**
  [`audit_agent.py`](ai-agents/audit_agent.py) — одной командой проверяет целостность
  всего governance-слоя (реестр/IPFS/Safe/Snapshot) с привязкой к статьям конституции.

Прозрачность: не верьте на слово — проверьте сами

Всё, что делает Public Trust DAO, можно прочитать и независимо проверить. На этой странице собраны ссылки на проверяемые артефакты и точные команды, которыми любой человек может убедиться, что история проекта не подменена, а опубликованные документы соответствуют их отпечаткам.

Что можно проверить

Каждый из этих артефактов публичен и самопроверяем. Ссылки ведут прямо в репозиторий — ничего не нужно принимать на веру.

  • Публичный реестр решений
    Append-only журнал с hash-chain: каждая запись ссылается на sha256 предыдущей, поэтому подмена истории становится заметна сразу. Без персональных данных заявителей — только псевдонимный case_id.

  • IPFS-манифест ключевых документов
    Для каждого ключевого файла — размер, sha256 и настоящий IPFS CIDv1, вычисленный прямо из содержимого. Документы доступны и проверяемы по контент-адресу, без доверия к какому-либо серверу.

  • Статусы непрерывной проверки (CI)
    На каждый коммит автоматически прогоняются проверки целостности реестра, IPFS-манифеста, конфигов governance, тестов смарт-контрактов и AI-агентов. «Красный» прогон — публичный сигнал, что что-то разошлось.

  • Смарт-контракты и их тесты
    Казна, целевой escrow, репутация, Governor и Timelock — с тестами, проверяющими именно конституционные свойства (никто не двигает средства единолично, лимиты, аварийная пауза). Только тестовые сети.

  • AI-агенты соблюдения конституции
    Восемь служебных модулей (Audit, Guardian, Fairness, Reputation, Housing, Documentation, Governance, Mediator) проверяют соблюдение рельс. ИИ служит, а не правит: находка — это сигнал, а не действие.

Реестр: 37 записей в hash-chain IPFS-манифест: 15 файлов с CID Контракты: 54 теста, только testnet Агенты: 8/8 с тест-инвариантами

Числа выше — справочные; источник истины всегда в репозитории. Текущий отпечаток всего реестра (head_hash) — это итоговая контрольная сумма всей цепочки решений:

Как проверить целостность самому

Все инструменты проверки написаны на чистом Python из стандартной библиотеки — без сторонних зависимостей и без выхода в сеть. Нужен только git и python3.

1

Скачайте репозиторий

Получите полную копию кода, документов и реестра решений.

git clone https://github.com/Public-Trust/public-trust-dao cd public-trust-dao
2

Проверьте реестр решений

Проверяет схему каждой записи, целостность hash-chain и соответствие индекса. Если хоть одна запись подменена — проверка падает с кодом 1.

python3 scripts/registry.py verify

Хотите убедиться, что защита работает? Измените любой символ в любой записи governance/registry/records/ и запустите снова — проверка покраснеет.

3

Проверьте IPFS-манифест документов

Пересчитывает размер, sha256 и IPFS-CID каждого файла из его содержимого и сверяет с манифестом. Любое незаметное изменение документа будет поймано.

python3 scripts/ipfs_manifest.py verify
4

Прогоните единый аудит

Аудит-агент объединяет проверки реестра, IPFS, конфигов Safe и Snapshot и связывает каждую со статьёй конституции, которую она защищает.

python3 ai-agents/audit_agent.py # вместе с тестами смарт-контрактов (нужны Node.js и npm): python3 ai-agents/audit_agent.py --with-contracts
5

Сверьтесь с публичным CI

Те же проверки автоматически идут на каждый коммит. Откройте вкладку Actions — зелёные прогоны означают, что на момент последнего коммита всё сходится.

Принципы этой страницы

Почему проверка устроена именно так

Доверие через проверку, не через обещание

Мы не просим верить на слово. Целостность доказывается математически — хэшами и контент-адресами, которые любой может пересчитать сам.

Ноль исходящих запросов

Сам этот сайт не подключает внешние шрифты, CDN, трекеры и даже значки статуса CI с чужих серверов — чтобы ничего не отправлять о посетителе. Поэтому статусы даны ссылками, а не картинками-бейджами.

Приватность заявителей

Открыты решения и операции, но не личности тех, кому помогают: в реестре — только псевдонимный case_id, без персональных данных.

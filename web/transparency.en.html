    What you can verify

    Each of these artifacts is public and self-verifiable. The links go straight into the repository — nothing has to be taken on trust.

    • Public decision registry
      An append-only, hash-chained log: every entry references the sha256 of the previous one, so any rewrite of history becomes immediately visible. No personal data about applicants — only a pseudonymous case_id.

    • IPFS manifest of key documents
      For each key file — size, sha256 and the real IPFS CIDv1 computed directly from its content. The documents are reachable and verifiable by content address, without trusting any single server.

    • Continuous integrity checks (CI)
      On every commit, automated checks re-verify the registry, the IPFS manifest, the governance configs, the smart-contract tests and the AI agents. A "red" run is a public signal that something has diverged.

    • Smart contracts and their tests
      Treasury, targeted escrow, reputation, Governor and Timelock — with tests that check the constitutional properties themselves (no one moves funds alone, limits, emergency pause). Test networks only.

    • Constitution-compliance AI agents
      Eight service modules (Audit, Guardian, Fairness, Reputation, Housing, Documentation, Governance, Mediator) check that the safety rails hold. The AI serves, it does not rule: a finding is a signal, not an action.

    How to verify integrity yourself

    All verification tools are written in plain Python from the standard library — no third-party dependencies and no network access. You only need git and python3.

    1

    Clone the repository

    Get a full copy of the code, the documents and the decision registry.

    git clone https://github.com/Public-Trust/public-trust-dao cd public-trust-dao
    2

    Verify the decision registry

    Checks the schema of each entry, the integrity of the hash-chain and that the index matches. If a single entry is altered, the check fails with exit code 1.

    python3 scripts/registry.py verify

    Want to confirm the protection works? Change any character in any record under governance/registry/records/ and run it again — the check turns red.

    3

    Verify the IPFS document manifest

    Recomputes the size, sha256 and IPFS CID of each file from its content and compares them with the manifest. Any silent change to a document will be caught.

    python3 scripts/ipfs_manifest.py verify
    4

    Run the unified audit

    The Audit agent bundles the registry, IPFS, Safe and Snapshot config checks and ties each one to the constitutional article it protects.

    python3 ai-agents/audit_agent.py # together with the smart-contract tests (needs Node.js and npm): python3 ai-agents/audit_agent.py --with-contracts
    5

    Cross-check the public CI

    The same checks run automatically on every commit. Open the Actions tab — green runs mean everything reconciled as of the latest commit.

    Principles of this page

    Why verification is built this way

    Trust through verification, not promises

    We don't ask you to take our word. Integrity is proven mathematically — with hashes and content addresses that anyone can recompute themselves.

    Zero outgoing requests

    This site itself loads no external fonts, CDNs, trackers — not even CI status badges from third-party servers — so it sends nothing about the visitor. That is why statuses are given as links, not badge images.

    Applicant privacy

    Decisions and operations are open, but not the identities of those helped: the registry holds only a pseudonymous case_id, no personal data.

## 0. Зачем проекту почта

- **Точка контакта.** Чтобы люди, СМИ, потенциальные хранители и партнёры могли
  написать проекту по понятному адресу, а не в личку случайного человека.
- **Доверие и прозрачность.** Публичный адрес = проект не прячется. Это прямо
  поддерживает миссию (общественное благо, всё открыто).
- **Технические нужды.** Регистрация на Snapshot/IPFS-пиннере/GitHub-организации,
  восстановление доступов, уведомления — всё привязывается к почте проекта, а не
  к личному ящику основателя.
- **Разделение ролей.** Почта проекта ≠ личная почта человека. Это поддерживает
  конституционный принцип «никто не владелец»: ящик принадлежит проекту и
  передаётся вместе с ролью, а не уходит с человеком.

## 1. Рельсы (читать ПЕРЕД настройкой)

1. **Никаких обещаний доходности в переписке.** Те же запреты, что и в
   [`PRINCIPLES.md`](PRINCIPLES.md): не инвестиция, не пирамида, не гарантии
   прибыли. Шаблоны ответов ниже уже это соблюдают.
2. **Пароль — только у оператора (и резерв у хранителей по решению).** Не в репо,
   не в чатах, не у ИИ-агента. Лучше — менеджер паролей.
3. **Включить 2FA** (двухфакторную аутентификацию) сразу при создании.
4. **Не публиковать чужие персональные данные.** Заявки и личные обращения —
   приватны (см. таблицу «что публично/приватно» в
   [`ESCROW-TARGETED-DISBURSEMENT.md`](ESCROW-TARGETED-DISBURSEMENT.md)).
5. **Минимум сторонних сервисов.** Почтовый провайдер — единственная внешняя
   зависимость; не подключать к ящику трекеры/аналитику/маркетинговые рассылки.
6. **Передаваемость.** Доступ к ящику должен быть восстановим без одного
   конкретного человека (резервные коды у нескольких хранителей) — почта
   принадлежит проекту, а не личности.

## 2. Варианты (сравнение)

Рекомендация по приоритету: **Вариант A (свой домен + почта) → B (ProtonMail на
их домене) → C (Gmail/прочее)**. Свой домен — лучший выбор для долговечности и
доверия; начать можно с B и переехать на A без смены адреса, если домен куплен
заранее.

| Критерий | A. Свой домен + почта | B. ProtonMail (домен @proton.me) | C. Gmail/обычная |
|---|---|---|---|
| Адрес | `hello@<домен>` | `publictrustdao@proton.me` | `...@gmail.com` |
| Доверие/бренд | высокое (свой домен) | среднее | низкое для «фонда» |
| Приватность | зависит от провайдера | высокая (шифрование, Швейцария) | низкая (сканирование) |
| Переезд адреса | не нужен (домен ваш) | потребует смены адреса | потребует смены адреса |
| Стоимость | домен ~10–15 $/год + почта | бесплатно / ~4 $/мес Plus | бесплатно |
| Кастомный домен | да | да (на платном тарифе) | да (Workspace, платно) |
| Сложность | средняя | низкая | низкая |

**Вывод для общественного блага с упором на прозрачность и приватность
обращений:** оптимально **A** (свой домен) с почтой через провайдера, уважающего
приватность (ProtonMail/Mailbox.org/Migadu). Если нужно стартовать сегодня и
бесплатно — **B** (ProtonMail), затем при покупке домена подключить его к Proton.

> Конкретный домен и финальный выбор — за оператором. Агент рекомендации дал;
> покупку домена и регистрацию ящика (телефон/оплата) делает человек.

## 3. Какие адреса завести

Начать с одного-двух, расширять по мере надобности (можно алиасами на один ящик):

| Адрес | Назначение |
|---|---|
| `hello@` (или `info@`) | общий вход: вопросы людей, СМИ, общее |
| `guardians@` | связь с хранителями миссии (внутренняя координация) |
| `security@` | сообщения об уязвимостях/злоупотреблениях (ответственное раскрытие) |
| `press@` | запросы СМИ (опционально, алиас на `hello@`) |

Минимум для старта: **`hello@`**. Остальные — алиасы, чтобы не плодить ящики.

## 4. Пошаговая инструкция оператору

### Вариант B (быстрый старт, ProtonMail) — рекомендуемый первый шаг

1. Открыть proton.me → Create account → выбрать `publictrustdao` (или согласованное имя).
2. Задать **надёжный пароль**, сохранить в менеджере паролей. Записать резервный код.
3. Включить **2FA** (Settings → Account → Security → Two-factor authentication).
4. В подпись письма вставить boilerplate из §6 ниже.
5. Сообщить агенту (через `comms/INBOX.md`) сам адрес (без пароля!) — агент
   проставит его в README/сайт/контактные тексты.

### Вариант A (свой домен) — для долговечности

1. Купить домен у регистратора (Namecheap/Porkbun/Cloudflare Registrar).
   Предложения по имени домена — в §5.
2. Завести почту у провайдера с поддержкой кастомного домена
   (ProtonMail Plus / Mailbox.org / Migadu / Fastmail).
3. Прописать у регистратора DNS-записи провайдера: **MX**, **SPF** (TXT),
   **DKIM** (TXT/CNAME), **DMARC** (TXT) — провайдер даёт готовые значения.
   Это защищает от подделки писем от имени проекта.
4. Включить **2FA**, сохранить пароль и резервные коды (резерв — у нескольких
   хранителей, см. рельс §1.6).
5. Создать алиасы из §3 на основной ящик.
6. Сообщить агенту адрес — агент проставит его в репозиторий.

> ⚠️ DNS-записи (домен, IP, MX-хосты) — публичны по своей природе и их можно
> коммитить как документацию инфраструктуры. **Пароли/секреты/токены — нет.**

# В каком порядке фонд помогает: кому раньше, когда денег на всех не хватает

> Это один из главных документов фонда. Он отвечает на простой и честный вопрос:
> если денег в общей казне меньше, чем просьб о помощи, **чью беду закрывать
> первой**. Документ вытекает из [конституции](CONSTITUTION.md) (статья 5
> «справедливое распределение») и [манифеста](MANIFESTO.md).
>
> Непонятные технические слова из этого документа объяснены простым языком в
> [глоссарии](GLOSSARY.md) — группа «Деньги и помощь».

## Главное — простыми словами

- **Сначала — спасение жизни и крыша над головой.** Если человеку прямо сейчас
  угрожает смерть или он вот-вот окажется на улице — это идёт впереди всего
  остального. Дальше — лечение и еда. Это базовые вещи, без которых человеку не
  выжить.
- **Помощь идёт по тяжести беды, а не по тому, кто человек.** Раньше получает не
  тот, кто «свой», известнее, активнее или дольше в проекте, а тот, чья нужда
  острее. Происхождение, репутация, статус участника на очередь **не влияют** —
  только сама беда (это прямой запрет дискриминации, статья 5.3 конституции).
- **Очередь — это про скорость рассмотрения, а не про поблажки в проверке.**
  Срочную беду смотрят первой, но это **не отменяет** обычных предохранителей:
  деньги всё равно идут по частям, есть лимиты, и каждую выплату проверяют
  несколько независимых людей. Высокий приоритет ускоряет, но не открывает
  «зелёный коридор» без проверки.
- **Когда беды одинаковой тяжести — делим честно и открыто.** Если две просьбы на
  одном уровне важности, правило, кого обслужить первым, заранее опубликовано и
  одинаково для всех — никаких закрытых решений «по знакомству».
- **Долгая польза для всех — в самом конце списка, но не выкинута.** Поддержка
  open source, науки, экологии, общей инфраструктуры важна, но она ждёт, пока
  закрыты живые человеческие беды. Сначала спасаем людей, потом вкладываемся в
  общее будущее.

## Порядок помощи — от самого срочного к менее срочному

| № | Кому помогаем раньше | Что это значит по-человечески |
|---|----------------------|-------------------------------|
| 1 | Угроза жизни | человеку прямо сейчас угрожает смерть |
| 2 | Угроза остаться без жилья | не дать человеку оказаться на улице, срочное жильё |
| 3 | Здоровье | лечение, лекарства, неотложная медицинская помощь |
| 4 | Еда | продукты, горячее питание |
| 5 | Дети | помощь детям и семьям с детьми |
| 6 | Пожилые люди | уход, сопровождение, базовые нужды |
| 7 | Люди с инвалидностью | поддержка людей с инвалидностью |
| 8 | Встать на ноги | работа, инструменты, выход из кризиса своими силами |
| 9 | Учёба | переобучение, новая профессия, цифровая грамотность |
| 10 | Долгая польза для всех | open source, наука, экология, общая инфраструктура |

## Как это работает на деле

1. **Мало денег — первыми идут те, чья беда тяжелее.** Чем выше человек в списке
   (ближе к «угрозе жизни»), тем раньше его просьбу берут в работу.
2. **Очередь не выключает защиту от обмана.** Все правила из
   [«Как фонд защищён от обмана и воровства»](ANTI-ABUSE.md) действуют всегда:
   деньги по частям, лимиты на крупные суммы, коллективная проверка. Срочность
   ускоряет рассмотрение, но не снимает проверки.
3. **Очередь зависит только от беды, а не от человека.** Репутация,
   происхождение, статус участника не двигают человека вверх или вниз — двигает
   только характер нужды (статья 5.3 конституции, запрет дискриминации).
4. **Равные по тяжести — обслуживаются по честному и открытому правилу.** Внутри
   одного уровня распределение прозрачно; правило, кого взять первым (например,
   кто раньше обратился), опубликовано и одинаково для всех.

## Технический раздел — для разработчиков

Этот порядок — спецификация для смарт-контрактов распределения и для
Fairness AI (ИИ-проверяющего справедливость). Здесь — машиночитаемое
представление, всё остальное выше изложено обычными словами.

Приоритеты кодируются целым числом `level` от `1` до `10`, где **меньшее число =
более высокий приоритет** (`level: 1` обслуживается раньше `level: 10`). Ключи
(`key`) стабильны и предназначены для использования в коде; подписи (`label`)
человекочитаемы и могут переводиться.

```json
{
  "priorities": [
    { "level": 1,  "key": "life_threat",         "label": "Угроза жизни" },
    { "level": 2,  "key": "housing_loss",        "label": "Угроза остаться без жилья" },
    { "level": 3,  "key": "medical",             "label": "Здоровье" },
    { "level": 4,  "key": "food",                "label": "Еда" },
    { "level": 5,  "key": "children",            "label": "Дети" },
    { "level": 6,  "key": "elderly",             "label": "Пожилые люди" },
    { "level": 7,  "key": "disability",          "label": "Люди с инвалидностью" },
    { "level": 8,  "key": "self_reliance",       "label": "Встать на ноги" },
    { "level": 9,  "key": "education",           "label": "Учёба" },
    { "level": 10, "key": "public_good",         "label": "Долгая польза для всех" }
  ]
}
```

При ограниченных средствах запросы с меньшим `level` обслуживаются раньше.
Приоритет не отменяет предохранителей из [`ANTI-ABUSE.md`](ANTI-ABUSE.md)
(поэтапные выплаты, лимиты, коллективная проверка) и не зависит от репутации,
происхождения или статуса участника — только от характера нужды (статья 5.3
конституции). Внутри одного уровня правила тай-брейка публикуются и одинаковы
для всех.

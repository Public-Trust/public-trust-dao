# КОНСТИТУЦИЯ СПРАВЕДЛИВОГО РАСПРЕДЕЛЕНИЯ — ПРИОРИТЕТ

> Нормативный документ. Определяет порядок приоритета при распределении помощи.
> Производный от [`CONSTITUTION.md`](CONSTITUTION.md) (статья 5) и
> [`MANIFESTO.md`](MANIFESTO.md). Этот порядок — спецификация для смарт-контрактов
> распределения и для Fairness AI.

## Порядок приоритета (от высшего к низшему)

| № | Приоритет | Краткое описание |
|---|-----------|------------------|
| 1 | Угроза жизни | непосредственная опасность для жизни человека |
| 2 | Угроза потери жилья | предотвращение бездомности, экстренное жильё |
| 3 | Медицинская помощь | лечение, лекарства, экстренная мед. помощь |
| 4 | Питание | продукты, горячее питание |
| 5 | Дети | помощь детям и семьям с детьми |
| 6 | Пожилые люди | уход, сопровождение, базовые нужды |
| 7 | Инвалидность | поддержка людей с инвалидностью |
| 8 | Восстановление самостоятельности | трудоустройство, инструменты, выход из кризиса |
| 9 | Образование | переквалификация, обучение, цифровая грамотность |
| 10 | Долгосрочная общественная польза | open source, наука, экология, инфраструктура |

## Машиночитаемое представление

Для использования в коде (смарт-контракты, Fairness AI) приоритеты кодируются
целым числом `1..10`, где **меньшее число = более высокий приоритет**.

```json
{
  "priorities": [
    { "level": 1,  "key": "life_threat",        "label": "Угроза жизни" },
    { "level": 2,  "key": "housing_loss",        "label": "Угроза потери жилья" },
    { "level": 3,  "key": "medical",             "label": "Медицинская помощь" },
    { "level": 4,  "key": "food",                "label": "Питание" },
    { "level": 5,  "key": "children",            "label": "Дети" },
    { "level": 6,  "key": "elderly",             "label": "Пожилые люди" },
    { "level": 7,  "key": "disability",          "label": "Инвалидность" },
    { "level": 8,  "key": "self_reliance",       "label": "Восстановление самостоятельности" },
    { "level": 9,  "key": "education",           "label": "Образование" },
    { "level": 10, "key": "public_good",         "label": "Долгосрочная общественная польза" }
  ]
}
```

## Правила применения

1. При ограниченных средствах запросы с меньшим `level` обслуживаются раньше.
2. Приоритет **не отменяет** проверок из [`ANTI-ABUSE.md`](ANTI-ABUSE.md):
   высокий приоритет ускоряет рассмотрение, но не отключает поэтапные выплаты,
   лимиты и коллективную проверку.
3. Приоритет **не зависит** от репутации, происхождения или статуса участника —
   только от характера нужды (статья 5.3 конституции, запрет дискриминации).
4. Внутри одного уровня приоритета распределение справедливо и прозрачно;
   правила тай-брейка публикуются и одинаковы для всех.
